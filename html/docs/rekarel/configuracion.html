Configuración

ReKarel tiene diferentes modos de opciones de configuración, a continuación se explican cada una

General

Vista

Controla la preferencia de vista de la aplicación.

  • Automático (Responsivo): La vista cambia dependiendo del tamaño de la pantalla
  • Vista de escritorio: Un formato con la pantalla divida horizontalmente entre el código y el mundo. Ideal para pantallas grandes y horizontales.
  • Vista de celular Un formato diseñado para pantallas pequeñas y verticales, como la de un celular.

Tema

Controla si la aplicación esta en cual tema.

  • Usar el tema del sistema: Usa el tema preferido por el navegador o sistema operativo. Si este no esta disponible, usa el Tema claro por defecto.
  • Tema claro: Fondos blancos e iluminados.
  • Tema oscuro: Fondos negros y oscuros.

Nota: Esta configuración no afecta al mundo de Karel, ya que este tiene su propio menú de configuración.

Editor del código fuente

Controla la configuración del editor de código

Tamaño de la fuente

Controla el tamaño de la fuente, el valor por defecto es 12pt.

Esta configuración se puede editar rápidamente presionando:

  • ctrl + K Para reducir el tamaño de la fuente
  • ctrl + L Para aumentar el tamaño de la fuente

Estilo

Controla el tema del editor, cambiando el resaltado de sintaxis, color de fondo, etc.

  • Karel.js: Tema inspirado por Karel.js.
  • ReKarel: Tema que utiliza los colores de la documentación de ReKarel.
  • Sepia: Disfruta de los tonos sepia.
  • Omi Karel: Regresa a la nostalgia azul con los colores de la versión original de Karel Omi.
  • Code: Inspirado en VSCode.

Los temas marcados con una luna cambian dependiendo de si se usa el Tema claro o el Tema oscuro

Cerrar paréntesis y llaves automáticamente

Si activado, al abrir un paréntesis o una llave (, [, {, se escribe automáticamente el cierre.

Si se tiene una selección al abrir una llave, esta es rodeada.

Editor del mundo

Controla el comportamiento del editor del mundo

Cambiar automáticamente el modo de selección

Si activado, cambia automáticamente al modo alternar si detecta entrada táctil.

Por defecto, esta activado

Ejecución

Controla el comportamiento de la ejecución.

Límite de instrucciones en modo normal:

La cantidad de instrucciones que se requieren para que se pase del modo normal de ejecución al modo rápido.

Valor defecto: 200,000

Advertencia: Colocar valores muy grandes aquí puede provocar que la aplicación tenga periodos extensos donde no la aplicación no es responsiva.
